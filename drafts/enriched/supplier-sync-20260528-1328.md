## Title

Supplier integration — pluggable adapters per supplier (Mouser, DigiKey, TME, Farnell, RS), daily Celery sync of prices/stock for the existing catalogue, and an on-demand `/components/lookup?mpn=` endpoint that pre-fills a new component by merging fields from all suppliers in priority order

## Problem / Context

Components already model historical `supplier_prices` and `supplier_stocks`, but no automated mechanism populates them. Today an operator must hand-enter prices and stock per supplier. We have 5 fixed suppliers (Mouser, RS Online, TME, Farnell, DigiKey). Their APIs vary in availability:

- **Self-serve, free, immediately available**: Mouser, DigiKey, TME, Farnell.
- **Requires sales-rep approval**: RS Online (App ID by request).

We have Celery + Celery Beat already wired in `app/infrastructure/celery_app.py` (empty `beat_schedule`) and running in `docker-compose.yml` as `celery_worker` + `celery_beat`. We need a daily job that walks the catalogue and refreshes prices/stock per supplier, plus a synchronous `GET /api/v1/components/lookup?mpn=...` endpoint that an operator (later, the FE) can hit to pre-fill the "Nuevo componente" form when typing an MPN.

This change is BACKEND-ONLY. No frontend work in this US (the existing `/components/new` form integration is a follow-up US once this endpoint is live and stable).

## Desired Outcome

1. A pluggable adapter pattern under `app/infrastructure/suppliers/` with one module per supplier exposing a common Protocol.
2. A Celery Beat job that runs daily at 03:00 UTC, iterates the existing components, calls each enabled supplier adapter, and upserts `supplier_prices` + `supplier_stocks` rows (append-only history).
3. A new `GET /api/v1/components/lookup?mpn=...` endpoint that walks the suppliers in priority order **Mouser → DigiKey → TME → Farnell → RS**, merges fields progressively (later suppliers only fill gaps left by earlier ones), and returns a single payload to pre-fill a new component.
4. Auditing table `supplier_sync_runs` capturing per-run telemetry.
5. Per-supplier enable/disable feature flag so the system can ship with RS disabled until credentials arrive.

## Acceptance Criteria (raw)

### Backend

#### Schema

- New table `supplier_sync_runs`:
  - `id` UUIDv4 PK
  - `supplier` varchar(32) NOT NULL — one of `('mouser','digikey','tme','farnell','rs')`
  - `started_at` timestamptz NOT NULL
  - `finished_at` timestamptz nullable
  - `components_processed` integer NOT NULL DEFAULT 0
  - `components_updated` integer NOT NULL DEFAULT 0
  - `errors_count` integer NOT NULL DEFAULT 0
  - `status` varchar(16) NOT NULL CHECK in `('running','success','partial','failed')` DEFAULT `'running'`
  - `error_summary` text nullable
  - `created_at` timestamptz
- New table `supplier_sync_errors` (rows produced when a single component fails for a supplier — bounded to last 90 days via a separate retention task; not in this US scope to prune):
  - `id` UUIDv4 PK
  - `run_id` UUIDv4 FK → `supplier_sync_runs.id` ON DELETE CASCADE
  - `component_id` UUIDv4 FK → `components.id` ON DELETE CASCADE
  - `supplier` varchar(32) NOT NULL
  - `error_code` varchar(64) NOT NULL (e.g. `RATE_LIMITED`, `NOT_FOUND`, `HTTP_5XX`, `PARSE_ERROR`, `AUTH_FAILED`)
  - `error_message` text NOT NULL
  - `occurred_at` timestamptz NOT NULL
- Migrations apply + reverse cleanly.

#### Domain & adapters

- Domain Protocol `SupplierAdapter` in `app/domain/repositories/supplier_adapter.py` exposing:
  - `code: str` (one of mouser/digikey/tme/farnell/rs)
  - `fetch_by_mpn(mpn: str) -> SupplierQuote | None` — used by both cron and lookup endpoint.
- Domain value object `SupplierQuote` with: `supplier`, `mpn`, `manufacturer`, `name`, `description`, `family_hint`, `datasheet_url`, `package`, `stock`, `price_breaks` (list of `{quantity, price_eur, currency_original, price_original}`), `supplier_sku`, `supplier_product_url`, `last_seen_at`.
- SQLAlchemy-side implementations live in `app/infrastructure/suppliers/{mouser,digikey,tme,farnell,rs}.py`. Each adapter:
  - Reads its own API key/secret from settings.
  - Honours a per-supplier rate limit via a Redis-backed token bucket (`app/infrastructure/rate_limit.py`).
  - Returns `None` cleanly on 404; raises a typed `SupplierError` subclass for transport/auth/parse failures.

#### MPN mapping per supplier (canonical reference)

Our canonical key is `Component.mpn`. Adapters MUST be able to look up by MPN regardless of the supplier's native vocabulary:

| Supplier | Search field | MPN field in response |
|---|---|---|
| Mouser | `SearchByPartRequest.mouserPartNumber` (accepts MPN too) | `ManufacturerPartNumber` |
| DigiKey | KeywordSearch v4 `keywords` or `manufacturerProductNumber` | `ManufacturerProductNumber` |
| TME | `Products/Search` `SearchPlain` | `OriginalSymbol` (NOT `Symbol` — that's TME's SKU) |
| Farnell | `keywordSearch` `keyword` | `translatedManufacturerPartNumber` |
| RS | `searchTerm` | `mpn` |

#### Settings

Add to `app/core/config.py`:
- `MOUSER_API_KEY: str | None`
- `DIGIKEY_CLIENT_ID: str | None`, `DIGIKEY_CLIENT_SECRET: str | None`, `DIGIKEY_OAUTH_TOKEN_URL: str` (default v4 prod URL)
- `TME_TOKEN: str | None`, `TME_APP_SECRET: str | None`
- `FARNELL_API_KEY: str | None`, `FARNELL_STORE_ID: str` (default `uk.farnell.com`)
- `RS_API_KEY: str | None`
- `SUPPLIER_SYNC_ENABLED_SUPPLIERS: list[str]` — comma-separated, default `mouser,digikey,tme,farnell` (RS off until credentials).
- `SUPPLIER_SYNC_DAILY_HOUR_UTC: int` (default `3`).
- `SUPPLIER_LOOKUP_CACHE_TTL_SECONDS: int` (default `900`).
- `SUPPLIER_LOOKUP_PRIORITY: list[str]` — default `mouser,digikey,tme,farnell,rs`.

When a supplier's keys are missing, that supplier is silently skipped in both cron and lookup (and a log line is emitted at INFO).

#### Endpoint — `GET /api/v1/components/lookup`

- Auth: `require_user`.
- Query parameters:
  - `mpn` (required, case-insensitive, trimmed; min length 3, max 60).
  - `force_refresh` (optional bool, default false) — bypasses the cache.
- Behaviour:
  1. Cache lookup in Redis under `supplier_lookup:{mpn_lower}` with TTL `SUPPLIER_LOOKUP_CACHE_TTL_SECONDS`. Skip if `force_refresh=true`.
  2. Iterate `SUPPLIER_LOOKUP_PRIORITY`, skipping disabled or unconfigured suppliers.
  3. For each supplier, call `fetch_by_mpn(mpn)`. On `None`, continue. On transport error, capture and continue (do not fail the whole request unless every supplier errors).
  4. Merge into a single `LookupResponse`: a higher-priority supplier wins for any non-null field; later suppliers only fill nulls.
  5. The `supplier_data` array preserves each supplier's raw quote (stock + price breaks + URL + SKU) for the FE to show.
  6. Returns 404 RFC-7807 if no supplier returned any data and at least one was consulted successfully (vs. 502 if all consulted suppliers errored).
- Response shape (`LookupResponse`):
  ```json
  {
    "mpn": "NBC12429FAR2G",
    "found": true,
    "fields": {
      "name": "...",
      "description": "...",
      "manufacturer": "...",
      "family_hint": "...",
      "datasheet_url": "...",
      "package": "...",
      "current_price_per_100_eur": 12.34
    },
    "supplier_data": [
      {
        "supplier": "mouser",
        "supplier_sku": "...",
        "supplier_product_url": "...",
        "stock": 1240,
        "price_breaks": [{"quantity": 1, "price_eur": "...", "price_original": "...", "currency_original": "USD"}]
      }
    ],
    "sources_consulted": ["mouser", "digikey"],
    "sources_succeeded": ["mouser", "digikey"],
    "missing_fields": ["package"]
  }
  ```
- 404 RFC-7807: `code = COMPONENT_MPN_NOT_FOUND`.
- 502 RFC-7807: `code = SUPPLIER_LOOKUP_UNAVAILABLE` (all suppliers failed transport-wise).

#### Daily sync job

- Celery task `app.infrastructure.tasks.supplier_sync.run_daily_sync()` registered in `celery_app.beat_schedule` under cron `0 SUPPLIER_SYNC_DAILY_HOUR_UTC * * *`.
- For each enabled supplier:
  1. Create a `supplier_sync_runs` row (`status='running'`).
  2. Fan out one Celery sub-task per chunk of N components (configurable, default 50) → `sync_supplier_chunk(supplier, component_ids)`.
  3. Each sub-task calls `fetch_by_mpn(component.mpn)`. On match, upserts the latest `supplier_prices` (append) and `supplier_stocks` (append) rows, and updates `component.last_supplier_sync_at`.
  4. On completion of all sub-tasks (chord), update the run row with totals and `status`.
- `acks_late=True`, `worker_prefetch_multiplier=1` already in `celery_app.py` — preserved.
- Rate limit: Mouser ≤ 30 req/min, DigiKey ≤ 1000/day, TME ≤ 20 req/min, Farnell ≤ default 60/min, RS ≤ default 60/min (all overridable via settings).

#### New tiny admin endpoint (read-only)

- `GET /api/v1/supplier-sync/runs?limit=50&supplier=mouser` — paginated last runs, ordered by `started_at DESC`. Auth `require_user`. Used later by an internal admin view (out of scope here, but the endpoint ships now).
- `GET /api/v1/supplier-sync/runs/{id}/errors?limit=200` — paginated errors for a given run.

#### Manual trigger (operator-only)

- `POST /api/v1/supplier-sync/runs?supplier=mouser` — enqueue an ad-hoc sync for one supplier. Returns the new `run_id` immediately. Used to test new credentials without waiting for 03:00 UTC.

### Tests

#### Backend (pytest, 80% gate)

- Unit:
  - Each adapter has contract tests against frozen fixture responses (`tests/fixtures/suppliers/{supplier}/by_mpn/*.json`) — happy match, no-match (404), transport error, parse error.
  - Lookup merge algorithm: M only, M+D fills gaps, all 5 mocked, all-null returns `found=false`, RS disabled is silently skipped.
  - Rate limit token bucket: under quota allows, over quota raises `RateLimitedError`.
- Integration:
  - `GET /components/lookup?mpn=...` — 200 happy, 404 not found, 502 all-failed, cache hit shape, `force_refresh=true` bypasses cache.
  - 401 without auth.
  - `GET /supplier-sync/runs` pagination + filter by supplier.
  - `POST /supplier-sync/runs?supplier=...` returns 202 + run id.
  - Daily sync task end-to-end against a mocked HTTP layer (httpretty / respx): updates `supplier_prices` + `supplier_stocks`, finalises the run row.

## Constraints / Notes

- **No frontend work** in this US. The `/components/new` form integration is a follow-up US.
- **RS Online** ships disabled by default; switch it on via `SUPPLIER_SYNC_ENABLED_SUPPLIERS` once we have the App ID.
- All adapters must convert prices to EUR (use the supplier's response currency + a cached daily FX rate; if FX unavailable, store `price_eur=null` and keep `price_original` + `currency_original`).
- **Idempotency**: the same MPN looked up twice within the cache TTL returns the cached payload. The daily sync upserts append-only rows — running it twice in the same day creates two history points, which is acceptable.
- All user-facing error messages in Spanish (RFC 7807 `detail`). All code/comments/logs in English.
