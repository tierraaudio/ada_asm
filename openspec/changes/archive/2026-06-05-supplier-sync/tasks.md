## 5. Infrastructure — adapters (one task block per supplier)

- [x] 5.1 Mouser adapter (`app/infrastructure/suppliers/mouser.py`): POST to `https://api.mouser.com/api/v2/search/partnumber?apiKey=...` with `SearchByPartRequest`; map response → `SupplierQuote`; rate-limit at 30 req/min. Fixtures: `hit.json`, `miss.json`, `auth_failed.json`. 9 unit tests + live smoke against real key with MPN `NBC12429FAR2G` returning a real EUR-quoted price break, all green.
- [x] 5.2 DigiKey adapter (`app/infrastructure/suppliers/digikey.py`): OAuth2 client_credentials grant against `/v1/oauth2/token` (form-urlencoded body), bearer-cached in-process with 30s headroom. KeywordSearch V4 at `POST /products/v4/search/keyword` with locale headers (Site=ES, Language=en, Currency=EUR) so prices come back EUR directly. Picks exact-MPN candidate from `Products[]`, walks `ProductVariations[]` to find one with non-empty `StandardPricing[]`, maps to our `SupplierPriceBreak` shape. Exposes `package` from the chosen variation's `PackageType.Name`. HTTP 429 surfaces as typed `SupplierRateLimitedError` so the 1000/day quota exhaustion path is unambiguous. Rate-limit bucket: 60 req/min. Fixtures: `auth_token.json`, `hit.json`, `miss.json`. 12 unit tests + live smoke against real key with MPN `NE555P` returning 26.735 stock + 8 EUR price breaks + datasheet URL + Tube package, all green.
- [x] 5.3 TME V2 adapter (`app/infrastructure/suppliers/tme.py`): NOT V1 HMAC. V1 is dead on this account regardless of "Key purpose" — all three options return `E_AUTHORIZATION_FAILED`. **V2 uses OAuth2 client_credentials** (`POST /auth/token` with HTTP Basic, 50-char username + 20-char password) → JWT bearer, then `GET /products/search` (returns `data.products.elements[]` with `symbol`, `manufacturer_symbols[]`, etc.) → `GET /products/data` (returns `data.elements[]` with `stock_quantity` and `prices.elements[]`). MPN resolved from `manufacturer_symbols[]` (NOT `symbol` — that's TME's SKU). EUR requested directly so FX is identity. In-process token cache with 30s expiry headroom. Rate-limit at 20 req/min (one bucket for both calls). Fixtures: `hit_search.json`, `hit_data.json`, `miss.json`, `auth_token.json`. 10 unit tests + live smoke against real key with MPN `NE555P` returning 17.997 stock + 8 EUR price breaks, all green.
- [x] 5.4 Farnell adapter (`app/infrastructure/suppliers/farnell.py`): GET `https://api.element14.com/catalog/products` with `term=any:<MPN>`, `responseGroup=large`, API key in `callInfo.apiKey`. Parses `keywordSearchReturn.products[]`; resolves MPN from `translatedManufacturerPartNumber` (exact-match selection across multi-candidate responses); reads stock from `stock.level` (fallback `inv`); price ladder from `prices[].from`/`cost`. Currency inferred from store host (`es/de/fr/it/pt/be/nl/at` → EUR; `uk/ie` → GBP; `newark.com` → USD); FX conversion only runs when the store isn't EUR. Builds best-effort product URL `https://<store>/_/dp/<sku>` (Basic responseGroup does not expose canonical URLs). Rate-limit at 100 req/min (well under the 2 req/sec Basic-tier ceiling). Fixtures: `hit.json`, `miss.json`, `auth_failed.json`. 11 unit tests + live smoke against real key with MPN `NE555P` returning 14.954 stock + 7 EUR price breaks, all green.

## 7. Daily sync — Celery task

- [x] 7.1 Task module `app/infrastructure/tasks/supplier_sync.py` exposing `run_daily_sync()` orchestrator (Beat-triggered) and `sync_one_supplier(code)` worker task. Per-supplier loop: creates `supplier_sync_runs` row → walks all components → calls adapter `fetch_by_mpn` → upserts the 4 fixed `qty_tier` rows in `supplier_prices` (mapping arbitrary supplier breaks to tiers via "largest break not exceeding tier") + appends `supplier_stocks` snapshot + bumps `components.last_supplier_sync_at` → on `SupplierError` inserts `supplier_sync_errors` row with typed `error_code` → finalises the run with `status ∈ {success, partial, failed}`. Async work wrapped with `asyncio.run` so the Celery sync worker process can drive the async adapters/DB. INSERT … ON CONFLICT DO UPDATE keeps same-day re-runs idempotent.
- [x] 7.2 Repository `app/infrastructure/repositories/supplier_sync_run_repository.py` with `list_recent(limit, supplier?)`, `get(run_id)`, `list_errors_for_run(run_id, limit)`. Reads only — writes happen inside the Celery task.
- [x] 7.3 Beat schedule entry `supplier-sync-daily` registered in `app/infrastructure/celery_app.py` via `crontab(hour=settings.supplier_sync_daily_hour_utc, minute=0)`. Worker + Beat now `imports=("app.infrastructure.tasks.supplier_sync",)` so tasks land on the registry at boot.
- [x] 7.4 Unit tests (`tests/unit/test_supplier_sync_logic.py`, 9 cases): tier-mapping picks largest break ≤ tier, single-bulk breaks (qty=2000 only) return None for every tier, DigiKey's 8-break ladder maps correctly, run finalisation status promotion across {empty, all-ok, mixed, all-errored, half-half}.
- [x] 7.5 Integration tests (`tests/integration/test_supplier_sync_task.py`, 4 cases against real Postgres): full ladder persistence + supplier auto-creation + `last_supplier_sync_at` bump; typed error → `supplier_sync_errors` row + `status=partial`; `None` quote writes nothing but doesn't error; same-day re-run idempotent via ON CONFLICT. Also a live Celery run against the 4 real suppliers (`mouser`, `digikey`, `tme`, `farnell`) on a seeded `NE555P` produced 4 `success` run rows with 1 update each.
